# 接口出入参对比分析

## 功能说明

对比分析新老链路接口本身的入参和出参。

---

## 入参对比

### 对比项

- **参数完整性**: 新链路是否包含所有老链路需要的参数
- **参数类型一致性**: 类型是否匹配
- **必填性一致性**: 是否兼容老调用方
- **新增参数**: 是否影响老调用方
- **默认值一致性**: 参数默认值是否保持兼容

### 输出格式

```
| 参数名 | 老链路类型 | 老链路必填 | 老链路默认值 | 新链路类型 | 新链路必填 | 新链路默认值 | 是否兼容 | 备注 |
|--------|-----------|-----------|-------------|-----------|-----------|-------------|---------|------|
```

### 分析要点

- 标记出缺失的参数(老链路有但新链路没有)
- 标记出新增的参数(新链路有但老链路没有)
- 标记出类型变化、必填性变化、默认值变化
- 评估每个变化对兼容性的影响

---

## 出参对比

### 对比项

- **返回字段完整性**: 老调用方需要的字段是否都有
- **字段类型一致性**: 类型是否匹配
- **字段含义一致性**: 同名字段含义是否相同
- **新增字段**: 是否影响老调用方解析
- **字段顺序**: 是否影响序列化/反序列化

### 输出格式

```
| 字段名 | 老链路类型 | 老链路含义 | 新链路类型 | 新链路含义 | 是否兼容 | 备注 |
|--------|-----------|-----------|-----------|-----------|---------|------|
```

### 分析要点

- 标记出缺失的字段(老链路有但新链路没有)
- 标记出新增的字段(新链路有但老链路没有)
- 标记出类型变化、含义变化
- 评估每个变化对兼容性的影响

---

## 兼容性评估

### 高风险

- 缺失关键参数
- 参数类型不兼容
- 缺失必需的返回字段
- 返回字段类型不兼容

### 中风险

- 新增必填参数
- 参数必填性变化
- 返回结构变化
- 字段含义变化

### 低风险

- 新增可选参数
- 新增返回字段
- 字段顺序变化(对JSON无影响)

---

## 分析方法

### 1. 数据提取

从JSON格式的出入参定义中提取:

```javascript
// 老链路入参示例
{
  "老链路入参": {
    "orderId": "string",
    "userId": "string",
    "amount": "number"
  }
}

// 新链路入参示例
{
  "新链路入参": {
    "orderId": "string",
    "userId": "string",
    "amount": "number",
    "currency": "string"  // 新增字段
  }
}
```

### 2. 字段对比

遍历所有字段,逐一对比:

- 字段名称
- 字段类型
- 字段必填性(根据JSON Schema或注解)
- 字段默认值

### 3. 差异识别

识别以下差异类型:

- **新增字段**: 新链路有,老链路没有
- **缺失字段**: 老链路有,新链路没有
- **类型变化**: 同名字段类型不同
- **必填性变化**: 同名字段必填性不同

### 4. 影响评估

对每个差异进行影响评估:

- 对调用方的影响
- 对下游服务的影响
- 对数据兼容性的影响
- 对业务逻辑的影响
