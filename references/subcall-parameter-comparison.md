# 子调用出入参对比分析

## 功能说明

对比分析新老链路接口内部子调用的入参和出参。

---

## 子调用识别

### 识别方法

首先识别新老链路的所有子调用:

- **子调用类型**: RPC调用、HTTP调用、数据库查询、缓存访问、消息队列操作等
- **调用信息**: 调用位置、被调用服务名称、被调用接口名称

### 输出格式

```
| 子调用编号 | 老链路位置 | 新链路位置 | 被调用服务 | 被调用接口 |
|-----------|-----------|-----------|-----------|-----------|
```

### 示例

```
| 子调用编号 | 老链路位置 | 新链路位置 | 被调用服务 | 被调用接口 |
|-----------|-----------|-----------|-----------|-----------|
| 1 | OrderService.createOrder:45 | NewOrderService.createOrderV2:52 | UserService | getUserInfo |
| 2 | OrderService.createOrder:67 | NewOrderService.createOrderV2:78 | PaymentService | createPayment |
| 3 | OrderService.createOrder:89 | - | InventoryService | deductStock |
```

---

## 子调用入参对比

### 输出格式

```
子调用: [服务名].[接口名]
| 参数名 | 老链路传入值/类型 | 新链路传入值/类型 | 是否一致 | 差异说明 |
|--------|-----------------|-----------------|---------|---------|
```

### 分析要点

- **参数值来源**: 硬编码、配置、上游传入
- **参数映射关系**: 是否一致
- **参数组装逻辑**: 是否等价
- **参数类型转换**: 是否存在隐式转换

### 示例

```
子调用: UserService.getUserInfo
| 参数名 | 老链路传入值/类型 | 新链路传入值/类型 | 是否一致 | 差异说明 |
|--------|-----------------|-----------------|---------|---------|
| userId | req.userId (string) | req.userId (string) | 一致 | 无变化 |
|字段 | "detail" (string) | "full" (string) | 不一致 | 查询字段范围变化 |
```

---

## 子调用出参对比

### 输出格式

```
子调用: [服务名].[接口名]
| 返回字段 | 老链路使用方式 | 新链路使用方式 | 是否一致 | 差异说明 |
|---------|--------------|--------------|---------|---------|
```

### 分析要点

- **字段提取**: 返回值的字段提取是否一致
- **处理逻辑**: 返回值的处理逻辑是否等价
- **异常处理**: 异常处理方式是否一致
- **空值处理**: null或空值的处理是否一致

### 示例

```
子调用: UserService.getUserInfo
| 返回字段 | 老链路使用方式 | 新链路使用方式 | 是否一致 | 差异说明 |
|---------|--------------|--------------|---------|---------|
| userName | 赋值给order.userName | 赋值给order.userName | 一致 | 无变化 |
| userLevel | 用于判断优惠等级 | 用于判断优惠等级 | 一致 | 无变化 |
| userType | 未使用 | 用于新增的会员类型逻辑 | - | 新增使用 |
```

## 子调用完整性对比

### 输出格式

```
| 子调用功能 | 老链路实现 | 新链路实现 | 是否覆盖 | 风险说明 |
|-----------|-----------|-----------|---------|---------|
```

### 分析要点

- **缺失子调用**: 识别新链路缺失的子调用
- **新增子调用**: 识别新链路新增的子调用
- **调用顺序变化**: 识别子调用顺序变化
- **影响评估**: 评估每个变化对业务逻辑的影响

### 示例

```
| 子调用功能 | 老链路实现 | 新链路实现 | 是否覆盖 | 风险说明 |
|-----------|-----------|-----------|---------|---------|
| 获取用户信息 | UserService.getUserInfo | NewUserService.getUserInfo | 部分覆盖 | 新接口返回更多字段 |
| 创建支付 | PaymentService.createPayment | NewPaymentService.createPaymentV2 | 完全覆盖 | 接口升级但行为一致 |
| 扣减库存 | InventoryService.deductStock | - | 未覆盖 | 高风险：缺少库存扣减逻辑 |
```

---

## 分析方法

### 1. 子调用识别

从JSON格式的子调用定义中提取:

```javascript
// 老链路子调用示例
{
  "老链路子调用": [
    {
      "服务名": "UserService",
      "接口名": "getUserInfo",
      "入参": {
        "userId": "string",
        "字段": "string"
      },
      "出参": {
        "userName": "string",
        "userLevel": "string"
      }
    }
  ]
}

// 新链路子调用示例
{
  "新链路子调用": [
    {
      "服务名": "NewUserService",
      "接口名": "getUserInfo",
      "入参": {
        "userId": "string",
        "字段": "string"
      },
      "出参": {
        "userName": "string",
        "userLevel": "string",
        "userType": "string"
      }
    }
  ]
}
```

### 2. 参数对比

对每个子调用进行参数对比:

- 对比入参字段名、类型、来源
- 对比出参字段名、类型、使用方式
- 识别参数映射和转换逻辑

### 3. 行为对比

对比子调用的行为:

- 是否幂等
- 是否有副作用
- 失败后的影响范围
- 重试机制

### 4. 依赖分析

分析子调用的依赖:

- 子调用之间的依赖关系
- 子调用与主流程的关系
- 子调用的失败对主流程的影响

---

## 风险评估

### 高风险

- 缺失关键子调用
- 子调用出入参不兼容
- 子调用行为发生变化
- 子调用失败影响增强

### 中风险

- 子调用顺序变化
- 子调用新增可选参数
- 子调用返回新增字段
- 子调用性能变化

### 低风险

- 子调用实现方式变化但结果等价
- 子调用新增不影响业务的字段
- 子调用性能提升
